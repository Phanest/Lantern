When we look at any web site on the internet, we are often able to discern the different groupings that the designers have put there for us (Buttons are parts of menus, hamburger menus hide the clutter, and the content has a certain look to it), because the site is not just a mishmash of different elements splattered all over, but has a certain flow to it we are able to have a nice experience on those sites. Yet for blind people some sites can be a real struggle, even though there is software that can help these people navigate the internet, its ability is limited, not because of faults in the software but in the way sites are created.
Current navigation software functions by reading the text in the elements (which sometimes might prove confusing), or by using a few helpers whose ability to provide the blind user with a specially tailored experience is quite limited. This is due to the fact that the elements this software uses are there to tell the machine how to render the site, they contain structural information which is not intended for humans to read. Lantern provides a way to enrich these sites and create a special experience for the blind users using RDFa annotations.

RDFa is a way to include machine-readable information that expresses relationships between people, places and other things. It is used to make search engines aware of useful things (ex. business phone numbers, band tours and basic information on people), link web pages among each other, and annotate web pages.
The last of which is our intention. There are many vocabularies that describe various things (music and bands, people and others). Yet we can as simply (should we find the existing vocabularies lacking) create our own. With RDFa we can organize concepts into entities, so that we can describe whether we are talking about a person, a phone number, a band or something else entirely. Each entity can contain multiple properties, which can be simple values or relationships with other entities.

Using RDFa, web page creators can design a special user interface for their blind users. This user interface can be similar to the experience that able-sighted users get when they visit the site, or it may be a different experience that caters more to the blind users visiting the site.

In order to introduce the reader to our framework, we have provided several use cases scattered around the site. Buttons and forms are introduced in the 'Log in' menu. Audio and Video elements are introduced in the 'Articles' menu To reach the menu, head back (by pressing the Ctrl + Up arrow twice) and selecting the menu.
